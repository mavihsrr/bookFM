Sticky Header Logo With Scroll-Scrubbed Scale

Scroll down to see the logo shrink smoothly. Scroll up to expand it back. This implementation uses only GPU-friendly transform and opacity updates with GSAP + ScrollTrigger scrub for ultra-fluid behavior and no layout thrashing.

Performance Rule 1

Never animate layout properties on scroll. Keep updates on compositor-friendly properties to avoid forced reflow.

Performance Rule 2

Tie animation progress to the scrollbar via ScrollTrigger scrub to avoid stutter from event-driven manual updates.

Performance Rule 3

Use will-change sparingly on frequently-updated elements only. Here, the logo and side groups are isolated for transform updates.

Performance Rule 4

Set transform origin and translateZ(0) to keep rasterization stable and avoid visual jitter at high scroll speed.

Performance Rule 5

Use linear easing for scroll-scrubbed timelines so interpolation matches scroll movement exactly.

Performance Rule 6

Avoid heavy work in scroll handlers. ScrollTrigger batches updates with requestAnimationFrame and internal optimizations.

Performance Rule 7

Keep sticky layer simple and isolated. Complex paint effects can reduce smoothness when compositing over long pages.

Performance Rule 8

Use responsive breakpoints without changing animation strategy; the same transform-only timeline works for mobile and desktop.

Performance Rule 9

Prefer deterministic ranges. Here animation completes in about 200px from top for predictable UX.
